Why openai library with Gemma?

The OpenAI API is a format specification, not a product. It defines how to structure chat requests and responses as JSON over HTTP. LM Studio implements this same spec for any local model. The openai Python library is just an HTTP client that speaks this format — it works with any server that follows the spec, including LM Studio running Gemma. No data ever leaves your computer.

What is the ReAct loop?

Reason + Act. main.py sends the conversation to Gemma. If Gemma decides to call a tool, it returns a tool_calls object (not text). main.py executes those tools against the graph, appends the results as role: "tool" messages, and calls Gemma again. This repeats up to 8 times until Gemma responds with no tool calls — that's the final answer.

What is native tool calling?

Gemma 4 understands tool schemas at the model level. We pass tools: [...] — JSON schemas describing each graph function — in the API request. The model decides when to call them and returns structured tool_calls data. No XML parsing, no regex, no hacks. The model natively understands it has tools and uses them.

What is the graph?

A directed graph stored in graph.json, loaded into NetworkX in memory. Nodes are Sean's values, goals, aesthetic commitments, tensions, influences, and projects — each with a weight (0.0–1.0, epistemic certainty) and description. Edges are relationships (SUPPORTS, TENSIONS_WITH, INFORMS…) with a grounding — the justification for why that connection holds. Calliope reads and writes this graph during every conversation.

What does Calliope actually do?

Before responding, she is required to call find_values and find_tensions. She then follows relevant threads with get_node, get_neighbors, search. She identifies real tensions between what you're asking and what your graph commits you to — then helps resolve them. If the conversation reveals something true not yet in the graph, she asserts it.

QAT — what does that mean?

Quantization-Aware Training. The model is compressed to use less memory (roughly 8–10GB instead of ~25GB for full precision), but unlike naive post-hoc quantization, QAT bakes the compression into training — so quality stays close to the full model. This is why a 12B model runs comfortably on a 24GB M4 MacBook Air.
